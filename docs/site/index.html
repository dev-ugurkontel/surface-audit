Safe security smoke tests for known URLs

Catch staging and preview regressions before they ship.

surface-audit runs a bounded set of web security checks against a known target and turns the result into CI-friendly output: console, JSON, HTML, Markdown, and SARIF. It shines when you want a deterministic gate for preview, staging, and pre-deploy environments.

Install from PyPI Read the README Use the recipes

Use it when you already know the URL. Reach for broader DAST tooling when you need crawling, authenticated scans, or exploit confirmation. 

$ surface-audit scan "$PREVIEW_URL" \
    --scope-host preview.example.com \
    --baseline reports/baseline.json \
    --fail-on HIGH \
    --output reports/preview.sarif \
    --format sarif

Preview-first

Built for staging, preview, and pre-deploy URLs where you want a repeatable security smoke test instead of a full crawling session.

Regression-focused

Baseline suppression and report diff workflows help teams answer “what changed?” after deployments, framework upgrades, and infrastructure edits.

MCP-safe

The optional MCP server stays behind an explicit host allow-list so agent-driven usage remains bounded and explainable.

Adoption paths

The same core scan works in local CLI use, CI gates, SARIF uploads, Markdown PR summaries, and MCP-driven workflows.

GitHub Action

- name: Run surface-audit
  uses: dev-ugurkontel/surface-audit@v1
  with:
    target: ${{ steps.preview.outputs.url }}
    scope-hosts: preview.example.com
    output: reports/surface-audit.sarif
    format: sarif
    fail-on: HIGH

MCP server

pip install "surface-audit[mcp]"
surface-audit mcp-serve \
  --allow-host staging.example.com \
  --allow-host preview.example.com

What belongs in core

The project intentionally optimizes for a narrow, dependable story rather than a huge vulnerability template catalog.

Use surface-audit when

  • you have a known target URL
  • you want CI-friendly output and `--fail-on` gating
  • you care about safe, deterministic surface checks
  • you want baseline/diff workflows for regressions

Reach for other tools when

  • you need crawling or spidering
  • you need authenticated browser or session flows
  • you want exploit confirmation
  • you need a very broad template library
